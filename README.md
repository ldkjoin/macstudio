# Mac Studio 库存监控

本地 Web 管理系统，用后台 Chrome 定时检测苹果中国官网 Mac Studio 指定配置是否有货。

## 启动

启动为后台进程，不会打开新的 Terminal 黑窗口：

```bash
./start.sh
```

重启会先停止当前服务，再重新构建并启动：

```bash
./restart.sh
```

停止：

```bash
./stop.sh
```

打开页面：

```text
http://127.0.0.1:4873
```

日志位置：

```text
logs/server.log
```

服务运行期间会自动保持 Mac 不进入系统睡眠，以保证定时检测和整点 Pushover 通知准时执行；屏幕仍会按系统设置正常关闭。停止服务后，Mac 会恢复正常睡眠行为。

## 页面功能

- 通过下拉选项配置芯片、处理器、统一内存、存储容量。
- 修改配置前需要先停止定时任务，保存后后续检测按新配置执行。
- 保存配置后会立即执行一次检测，避免结果面板显示旧型号。
- 检测结果会校验精确配置 URL、配置汇总和购买流程继续按钮。
- 定时检测全天持续运行，页面可分别配置上班时间频率和下班时间频率，默认 09:00-18:00 每 5 分钟检测一次，其余时间每 10 分钟检测一次。
- Pushover 是唯一通知通道，可配置一个 Application API Token 和多个 User Key / Group Key 接收人。
- 默认每天 09:05、14:05、22:05 通过 Pushover 普通通知发送服务状态总结，页面可调整时间或关闭总结。
- 检测到有货时，会向所有 Pushover 接收人发送 emergency priority 强提醒，并按页面配置的推送条数、重复间隔和过期时间执行。
- Pushover 接收人保存时会自动去重；测试通知会向所有已保存接收人发送普通 Pushover，推送条数与页面配置保持一致。
- 检测历史会记录每次检测的完整型号配置、判断原因、通知摘要和通知详情。

## Pushover 配置说明

- API Token：在 Pushover 后台创建 Application 后获得，一个项目通常只需要一个 Token。
- User Key：个人账号首页展示的接收 Key。
- Group Key：Pushover Delivery Group 的接收 Key，可像 User Key 一样添加到接收人列表。
- 费用：截至 2026-05-17，Pushover 官方价格为 30 天试用，之后个人版每个平台 4.99 美元一次性购买；普通账户通过 API 每月可免费发送 10,000 条消息，足够本项目常规使用。
